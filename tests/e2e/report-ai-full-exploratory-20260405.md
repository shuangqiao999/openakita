## 第十一部分：发现的问题汇总

### 问题 #1: 长文本输入后空响应 [CRITICAL]

- **现象**: T9 发送~600字的项目需求文档，SSE 流返回 0 字节文本
- **严重程度**: CRITICAL
- **影响**: 用户发送长消息时可能收不到回复
- **日志位置**: `tests/e2e/_test_log_20260405.jsonl` T9 条目；后端日志可能有对应的 LLM 请求超时
- **可能原因**: (1) LLM 调用工具后输出被吞；(2) 流式响应中工具调用结果未正确拼接到 text_delta
- **建议修复**: 检查 `chat.py` 中当 LLM 使用工具（如代码执行）生成长回复时，工具输出是否正确转为 text_delta 事件

### 问题 #2: 跨请求后会话上下文丢失 [CRITICAL]

- **现象**: T21 使用同一 conversation_id 发送总结请求，系统却回复"这是本次会话中的第一条消息"
- **严重程度**: CRITICAL
- **影响**: 用户在间隔几分钟后继续对话时，之前的上下文可能完全丢失
- **日志位置**: `tests/e2e/_test_log_20260405.jsonl` T21 条目
- **可能原因**: (1) Agent Pool 在空闲时回收了会话实例，新请求创建了新实例但未加载历史；(2) 中间经过了新会话(conv_id2)的 T18-T20，可能影响了 pool 中的会话状态
- **建议修复**: 检查 `AgentInstancePool` 的会话回收策略，确保 `get_or_create()` 在重新创建实例时正确加载 `session_manager` 中的历史

### 问题 #3: 图片附件尺寸限制无友好提示 [MEDIUM]

- **现象**: T23 发送 1x1 像素图片，LLM API 返回 400 错误，但系统返回空响应而非友好提示
- **严重程度**: MEDIUM
- **影响**: 用户发送过小图片时看不到任何回复
- **日志位置**: `tests/e2e/_test_log_20260405.jsonl` T23 条目
- **错误详情**: `InternalError.Algo.InvalidParameter: height:1 or width:1 must be larger than 10`
- **建议修复**: 在 `chat.py` 或 LLM 客户端层捕获图片尺寸错误，向用户返回友好提示如"图片尺寸过小，请使用宽高大于10像素的图片"

### 问题 #4: 组织命令/广播 API 文档不明确 [LOW]

- **现象**: 使用 `{"instruction": "..."}` 和 `{"message": "..."}` 作为 body 字段，实际需要 `{"content": "..."}`
- **严重程度**: LOW（不是代码 bug，是 API 文档/命名一致性问题）
- **位置**: `src/openakita/api/routes/orgs.py` L574-577, L659-661
- **建议**: 考虑在 API 文档中明确标注参数名，或添加 Pydantic 模型约束

### 问题 #5: `delegate_parallel` 工具缺少 context 参数 [HIGH]

- **现象**: LLM debug 日志显示 `delegate_to_agent` 有 `context` 参数，但 `delegate_parallel` 没有
- **严重程度**: HIGH
- **影响**: 并行委托任务时，子 Agent 可能缺少必要的上下文信息
- **日志位置**: `tests/e2e/_audit_result_20260405.txt` 第92行
- **建议修复**: 在 `delegate_parallel` 的工具定义中添加 `context` 参数（与 `delegate_to_agent` 一致）

### 问题 #6: 记忆提取器频繁失败 [HIGH]

- **现象**: 测试期间 45 条 ERROR 日志，全部是 `Memory Extractor/Encoder` 调用 LLM 失败
- **严重程度**: HIGH
- **影响**: 对话后的自动记忆提取/编码不工作，长期记忆无法积累（记忆列表为空 0 条）
- **日志位置**: 后端日志 L3574-L4292，错误模式 `ReadError(ReadError(''))`
- **可能原因**: LLM 端点在高并发下连接被重置（主对话流和后台记忆提取竞争同一端点）
- **建议修复**: (1) 为记忆提取配置独立的 LLM 端点或降低并发；(2) 添加重试机制；(3) 考虑请求队列限流

### 问题 #7: System Prompt Budget 截断警告 [MEDIUM]

- **现象**: 多条 Budget 截断警告：`soul: 3468 -> 3201 tokens (budget: 1800, truncated via end)`

### 问题 #9: WebSocket 代理连接中断 [LOW]

# 系统状态

## 概念

系统状态页面是 OpenAkita 的健康仪表盘，让你一眼看清所有关键组件的运行情况——服务是否正常、模型是否连通、通道是否在线。当遇到问题时，这是你的第一个排查入口。

---

## 状态面板

[打开系统状态](/web/#/status)

### 服务连接状态

显示 OpenAkita 核心服务的运行状态：

| 指标 | 说明 |
|------|------|
| 服务状态 | 运行中（running）或已停止（stopped） |
| 启动时间 | 服务持续运行时长 |
| API 地址 | 当前 API 监听地址和端口 |

### LLM 端点健康

显示已配置的 LLM 端点列表及其连接状态：

- 端点名称和模型
- 连接状态（在线 / 离线 / 超时）
- 最近一次调用的响应时间

👉 有问题？前往 [LLM 配置](/web/#/config/llm) 检查端点设置。

### IM 通道状态

列出所有已配置的消息通道及其连接情况：

- 通道类型（Telegram / 飞书 / 钉钉 / 企业微信等）
- 连接状态（已连接 / 未连接 / 认证失败）
- 消息收发统计

👉 有问题？前往 [消息通道](/web/#/im) 检查通道配置。

### 技能概览

- 已启用技能数量
- 外部技能数量（来自工作区或技能商店）
- 技能加载是否正常

### 记忆状态

- 记忆条目总数
- 搜索后端类型（FTS5 / ChromaDB / API Embedding）
- 最近一次整合时间

### 系统信息

| 字段 | 示例 |
|------|------|
| 操作系统 | Windows 10 / Ubuntu 22.04 / macOS 14 |
| Python 版本 | 3.11.8 |
| OpenAkita 版本 | v1.26.5 |
| 数据目录 | `data/` |

## 常见问题排查

### 服务无法启动

1. 检查端口是否被占用：确认 8000 端口（默认）没有被其他程序使用
2. 查看日志输出：启动时的错误信息通常能直接定位问题
3. 检查依赖：运行 `pip install -e ".[dev]"` 确保所有依赖安装完整

### LLM 端点连接失败

1. 确认 API Key 是否正确填写 — 前往 [LLM 配置](/web/#/config/llm) 检查
2. 确认网络是否能访问对应 API（国内访问 OpenAI 可能需要代理）
3. 检查端点 URL 是否正确，特别是使用自部署模型时
4. 查看状态面板中的具体错误信息

### IM 通道断开连接

1. 检查 Bot Token 是否有效且未过期
2. 确认 Webhook URL 是否可从外网访问（Telegram、飞书等需要公网回调）
3. 重新启用通道：在 [消息通道](/web/#/im) 中关闭再开启
4. 查看通道的错误日志，常见原因包括 Token 失效、网络中断、权限不足

::: tip 提示
大多数连接问题可以通过「重启服务」解决。如果问题持续，建议在 [GitHub Issues](https://github.com/openakita/openakita/issues) 中附上日志反馈。
:::

# Agent 管理

## 什么是 Agent 管理

Agent 管理是创建、编辑和组织 Agent 的中心页面。每个 Agent 有独立的身份、技能范围和行为偏好——就像给团队成员分配岗位和权限。

[打开 Agent 管理](/web/#/agent-manager)

## 创建 Agent

点击页面右上角 **「+ 新建 Agent」** 按钮，填写以下信息：

### 基本信息

| 字段 | 说明 | 示例 |
|------|------|------|
| **名称** | Agent 的显示名称 | 文档助手 |
| **描述** | 一句话说明职责，编排器据此做路由决策 | 专注于 Office 文档处理与排版 |
| **图标** | 选择或上传一个图标 | 📄 |
| **颜色** | 在组织编排和面板中的标识色 | #4A90D9 |
| **分类** | 归属的功能类别 | 办公效率 |

### 技能模式

控制该 Agent 可使用哪些技能：

| 模式 | 效果 |
|------|------|
| **all（全部）** | 该 Agent 可使用系统中所有已启用的技能 |
| **include（仅包含）** | 只能使用你勾选的技能，其余禁用 |
| **exclude（排除）** | 使用所有技能，但排除你勾选的 |

::: tip 何时使用 include 模式
当你希望 Agent 高度专注时使用。例如"浏览器 Agent"只需要网页浏览和截图技能，排除文件写入等可以避免误操作。
:::

### 自定义提示词

为 Agent 编写专属的系统提示词（System Prompt），定义其角色、行为规范和约束：

```
你是一名专业的数据分析师。
- 优先使用 Python pandas 进行数据处理
- 输出结果时附带数据可视化图表
- 涉及敏感数据时提醒用户确认
```

### 模型偏好

为该 Agent 指定专属模型。未设置时使用全局默认模型。

- **首选模型** — 优先使用的 LLM（如 claude-sonnet-4-20250514）
- 可在 [LLM 端点配置](/web/#/config/llm) 中管理可选模型列表

## 预设 Agent

系统内置以下预设 Agent，覆盖常见场景：

| 预设 Agent | 技能模式 | 特长 |
|-----------|---------|------|
| **default** | all | 通用对话、任务规划 |
| **office-doc** | include: 文档相关 | Word、PPT、Excel 处理 |
| **code-assistant** | include: 代码相关 | 编程、调试、代码审查 |
| **browser-agent** | include: 浏览器 | 网页浏览、截图、信息采集 |
| **data-analyst** | include: 数据相关 | 数据清洗、分析、可视化 |

预设 Agent 可直接使用，也可以复制后修改为自己的版本。

## 导入与导出

### 导出

- **单个导出** — 点击 Agent 卡片右上角菜单 → 导出为 JSON
- **批量导出** — 勾选多个 Agent → 工具栏「导出选中」

导出文件包含 Agent 的完整配置（名称、提示词、技能模式等），不含运行时数据。

### 导入

- 点击 **「导入 Agent」** → 选择 JSON 文件
- 名称冲突时可选择覆盖或重命名
- 支持从 [Agent Store](/web/#/agent-store) 一键安装

### 批量操作

勾选多个 Agent 后，工具栏支持：

- **批量删除** — 移除选中的 Agent
- **批量启用 / 禁用** — 临时停用某些 Agent
- **批量修改分类** — 统一归类

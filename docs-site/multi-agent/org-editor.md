# 组织编排

## 什么是组织编排

组织编排是一个可视化画布，让你像画组织架构图一样**拖拽搭建 AI 团队**。每个节点代表一个 Agent、Bot 或项目，连线表示协作与汇报关系。

如果多 Agent 是"团队"，组织编排就是"团队架构图编辑器"。

## 打开组织编排

[打开组织编排](/web/#/org-editor)

::: tip 前置条件
需要先在 [Agent 配置](/web/#/config/agent) 中开启多 Agent 模式，组织编排入口才会出现在侧边栏。
:::

## 画布操作

组织编排基于 React Flow 构建，支持以下操作：

| 操作 | 方式 |
|------|------|
| **平移画布** | 按住空白区域拖拽 |
| **缩放** | 鼠标滚轮，或使用左下角缩放控件 |
| **选中节点** | 单击节点 |
| **移动节点** | 拖拽节点到目标位置 |
| **连接节点** | 从节点边缘的端口拖出连线到目标节点 |
| **删除** | 选中后按 `Delete` 键 |
| **多选** | 按住 `Shift` 框选 |

## 节点类型

### Agent 节点

代表一个 AI 智能体。双击可编辑属性：

- **名称与图标** — 自定义显示信息
- **技能配置** — 指定该 Agent 可用的技能集
- **模型偏好** — 为该 Agent 指定专属 LLM
- **自定义提示词** — 设置角色定义和行为约束

### Bot 节点

代表一个外部机器人或 IM 通道连接。用于标注"这个 Agent 通过哪个 Bot 对外服务"。

### Project 节点

代表一个项目或任务分组。多个 Agent 可归属到同一个 Project，便于管理和可视化。

## 侧边面板

画布右侧有三个可折叠面板：

### Inbox 收件箱

显示当前 Agent 团队收到的待处理消息和任务请求，可直接在此分配给指定 Agent。

### Project Board 项目看板

以看板形式管理任务：待办 → 进行中 → 已完成。拖拽卡片即可更改状态。

### Chat 聊天面板

与选中的 Agent 直接对话，测试其行为或下达指令。

## 创建一个 Agent 团队

**快速上手示例：**

1. 在画布空白处右键 → **添加 Agent 节点**
2. 双击节点，设置名称为"调研助手"，技能选择浏览器相关
3. 再添加一个 Agent 节点，命名为"文档助手"，技能选择文档处理
4. 添加一个 Project 节点，命名为"竞品调研"
5. 从 Project 拖出连线到两个 Agent，建立归属关系
6. 从"调研助手"拖出连线到"文档助手"，表示调研结果会流转给文档助手

完成后，当用户发送与"竞品调研"相关的任务时，编排器会自动按此结构协调执行。

## 保存与同步

- 编排结果**自动保存**到工作区
- 修改会实时同步到运行中的编排器
- 可通过 [Agent 管理](/web/#/agent-manager) 查看和微调每个 Agent 的详细配置

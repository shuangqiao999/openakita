# 组织编排 — 功能介绍

> 用一个人的力量，运营一家公司。

---

## 这是什么？

OpenAkita 的**组织编排**功能，让你像搭积木一样构建自己的 AI 公司组织架构。

在传统模式下，你只能让主 Agent 临时调度子 Agent 执行任务。而组织编排不同——你可以编排一整套持久运行的"虚拟公司"，每个岗位都是一个独立的 AI Agent，它们有自己的身份、职责、记忆和定时任务，能够自主通信、协作、汇报，甚至主动发现问题。

**简单理解**：你是老板，定好公司架构和制度，下达指令后，这些 AI 员工就会按照分工自行运转。

---

## 核心特点

### 可视化组织架构编排

在拖拽式的编排画布上，你可以：

- **创建岗位节点** — 为每个岗位设定名称、目标和背景
- **建立关系连线** — 上下级关系、同级协作、咨询关系
- **分配部门** — 技术部、产品部、市场部……自由定义
- **从模板快速开始** — 内置创业公司、软件工程团队、内容运营团队三套模板，一键创建

编排完成后切换到「实况模式」，可以实时观察所有节点的运行状态、通信流和任务进度，节点会有呼吸灯效果、连线上有数据流粒子动画。

### 灵活的 Agent 配置

每个岗位节点可以：

- **引用已有 Agent** — 拖入你在 Agent 管理页创建的任何 Agent 副本
- **创建专属 Agent** — 为该岗位量身定制
- **四级身份配置** — 从零配置到完全自定义身份文件，按需渐进：
  - 零配置：直接使用引用 Agent 的原始配置
  - 写岗位说明（ROLE.md）：定义具体岗位职责
  - 覆盖行为规范：定制特殊行为准则
  - 完全独立身份：独立的人格和行为准则
- **独立工具配置** — 每个岗位可以额外携带或排除特定的 MCP 工具服务

### 自主通信与协作

组织内的 Agent 拥有完整的通信能力：

- **发消息、分配任务、汇报工作、上报问题** — 按组织架构自主路由
- **查看组织架构** — Agent 知道公司有哪些部门和同事
- **搜索合适的同事** — 不知道找谁时，描述需求让系统匹配
- **读写共享记忆** — 组织级公告、部门级协作上下文、个人工作笔记
- **查阅公司制度** — 遇到流程问题时查制度文件

### 三级共享记忆

类似公司的知识管理体系，分三个层级：

| 层级 | 类比 | 谁能看 | 用途 |
|------|------|--------|------|
| 组织级 | 公司公告栏 | 全员 | 战略决策、全局规则、重大进展 |
| 部门级 | 部门群聊 | 部门内 | 部门协作上下文、内部约定 |
| 节点级 | 个人笔记 | 仅自己 | 工作笔记、处理经验 |

Agent 执行任务时会自动获取相关记忆作为上下文，不需要你手动传递信息。

### 心跳驱动 — AI 公司自己会动

开启心跳功能后，组织会按设定的间隔自主运转：

- **定时心跳** — CEO 定时审视全公司状况，发现问题主动调配资源
- **晨会机制** — 可设定每工作日上午自动开"晨会"，各部门汇报进展
- **节点定时任务** — 每个岗位可以有自己的定期工作：
  - DevOps 每 30 分钟巡检服务器
  - 内容运营每天检查发布排期
  - SEO 每天检查关键词排名
  - PM 每周一汇总项目进度

### 动态扩编

运行过程中，Agent 可以自主申请调整人员配置：

- **克隆** — 某岗位忙不过来？申请加人手（克隆同岗位 Agent）
- **招募** — 发现缺少某种能力？申请新增岗位
- **裁撤** — 临时岗位任务完成后自动裁撤

所有人事变动都可以设置审批流程：自动审批、上级审批或用户审批。

### 统一消息中心

所有组织事件汇聚到右侧消息侧边栏，按 6 个优先级分级展示：

| 颜色 | 级别 | 示例 |
|------|------|------|
| 灰色 | 信息 (info) | "前端工程师A 完成了登录页开发" |
| 蓝色 | 通知 (notice) | "晨会纪要已生成" |
| 黄色 | 预警 (warning) | "组织 token 消耗已达预算 60%" |
| 橙色 | 操作 (action) | "法务顾问遇到无法判断的合规问题 [介入]" |
| 紫色 | 审批 (approval) | "CTO 申请克隆 1 名后端工程师 [批准/拒绝]" |
| 红色 | 警报 (alert) | "前端工程师B 连续 3 次任务失败，已自动冻结" |

需要审批的事项可以直接在消息栏内操作，无需跳转。

### IM 通道推送

可以将组织消息推送到你的飞书、钉钉、Telegram 等 IM 通道：

- 每条审批消息携带唯一编号（如 `#A12`）
- 直接在 IM 中回复 `#A12 批准` 或 `#A12 拒绝` 即可完成审批
- 支持免打扰时段设置

### 制度管理

像真正的公司一样建立制度体系：

- 组织总则 / 员工手册
- 各类工作流程（审批流程、代码评审流程、内容发布流程……）
- 部门规范和 SOP
- Agent 遇到流程问题会自动查制度，不会自行猜测

从模板创建的组织会附带配套的制度文件，你也可以自由添加和修改。

### 冻结与管控

如果某个 Agent 表现不佳：

- **冻结** — 上级 Agent 或你可以冻结任何节点，暂停其一切活动，保留数据和记忆
- **自动冻结** — 连续多次任务失败时自动冻结并通知
- **解冻** — 问题解决后随时恢复

## 怎么使用？

### 快速开始

1. 在左侧菜单进入**「组织编排」**页面（需开启多 Agent 模式）
2. 点击**「从模板创建」**，选择一套模板（推荐先试「创业公司」）
3. 在编排画布上浏览组织架构，按需调整岗位、连线和配置
4. 点击**「启动」**按钮激活组织
5. 切换到实况模式，观察组织运行状态

### 下达命令

有多种方式与组织互动：

- **聊天窗口**: 切换到组织模式，或输入 `@org:公司名 你的指令`
- **指定节点**: 输入 `@org:公司名/CTO 你的指令` 直接对 CTO 下命令
- **部门广播**: 输入 `@org:公司名/技术部 你的指令`
- **实况模式**: 在编排画布上点击任意节点直接对话

### 查看进展

- **实况模式**: 画布上实时显示节点状态、通信流、任务进度
- **消息侧边栏**: 右侧铃铛图标展开，查看所有组织事件和审批请求
- **IM 推送**: 在你的手机 IM 上随时接收关键通知

### 审批管理

当 Agent 申请扩编、提议制度或请求人工介入时：

- **在消息侧边栏**: 直接点击「批准」或「拒绝」
- **在 IM 中**: 回复 `#编号 批准` 或 `#编号 拒绝`（可附加原因）

## 内置模板

### 创业公司

16 个岗位，涵盖技术、产品、市场、行政四大部门：

```
CEO（首席执行官）
├── CTO（技术总监）
│   ├── 架构师
│   ├── 全栈工程师 A
│   ├── 全栈工程师 B
│   └── DevOps 工程师
├── CPO（产品总监）
│   ├── 产品经理
│   └── UI 设计师
├── CMO（市场总监）
│   ├── 内容运营
│   ├── SEO 专员
│   └── 社媒运营
└── CFO（财务总监）
    ├── HR（人力资源）
    └── 法务顾问
```

附带预置的通信规范、任务管理规范和扩编制度。DevOps 每 30 分钟巡检，PM 每周出项目进度周报，CFO 每周统计 token 消耗。

### 软件工程团队

10 个岗位，专注于软件研发场景：

```
技术负责人
├── 前端组长
│   ├── 前端开发 A
│   └── 前端开发 B
├── 后端组长
│   ├── 后端开发 A
│   └── 后端开发 B
├── QA 工程师
├── DevOps 工程师
└── 技术文档
```

### 内容运营团队

7 个岗位，适合内容创作和运营场景：

```
主编
├── 策划编辑
│   ├── 文案写手 A
│   ├── 文案写手 B
│   └── 视觉设计
├── SEO 优化师
└── 数据分析
```

## 适用场景

- **个人创业者**: 用 AI 编排一整个公司团队，从战略到执行全覆盖
- **内容创作者**: 编排选题、写作、优化、发布的全流程团队
- **软件开发者**: 搭建完整的研发团队进行开发、测试、部署协作
- **项目管理**: 编排项目组，自动跟踪进度、生成报告、识别风险
- **运维监控**: 编排值守团队，定时巡检、异常告警、自动处理

---

## 与原有功能的关系

组织编排是**新增模式**，原有的功能全部保留：

- **单 Agent 对话** — 照常使用
- **自由派发多 Agent** — 原有的协作动态面板照常运行
- **组织编排** — 新增的持久化组织模式

三种模式互不干扰，你可以在聊天窗口随时切换目标。
